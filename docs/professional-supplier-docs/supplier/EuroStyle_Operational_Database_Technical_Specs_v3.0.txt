================================================================================
                                SYSTEM OVERVIEW
================================================================================

The EuroStyle Operational Management System provides comprehensive enterprise
resource planning for fashion retail operations across Europe. Built on modern
ClickHouse columnar database technology, the system handles high-volume
operational data with real-time analytics capabilities.

CORE CAPABILITIES:
- Customer lifecycle management (1070+ active customers)
- Product catalog management (530+ fashion products)  
- Order processing and fulfillment (600+ orders)
- Multi-location inventory tracking (23,409+ inventory records)
- Store operations management (35 store locations)
- Campaign management and marketing automation
- European geography and fashion calendar integration

================================================================================
                               DATABASE SCHEMA
================================================================================

DATABASE NAME: eurostyle_operational
TABLES: 10 operational tables
TOTAL RECORDS: 26,764 (as of October 15, 2024)
ENGINE: ClickHouse MergeTree() with optimized sorting keys

TABLE SPECIFICATIONS:

1. CUSTOMERS (1,070 records)
   Primary Key: customer_id
   Engine: MergeTree() ORDER BY customer_id
   Description: Complete customer master data with European GDPR compliance
   Key Fields: customer_id, first_name, last_name, email, country, loyalty_tier

2. PRODUCTS (530 records)
   Primary Key: product_id
   Engine: MergeTree() ORDER BY product_id  
   Description: Fashion product catalog with seasonal and pricing information
   Key Fields: product_id, name, category, brand, price_eur, season

3. ORDERS (600 records)
   Primary Key: order_id
   Engine: MergeTree() ORDER BY order_id
   Description: Customer orders with complete lifecycle tracking
   Key Fields: order_id, customer_id, order_date, status, total_amount_eur

4. ORDER_LINES (1,068 records)
   Primary Key: order_line_id
   Engine: MergeTree() ORDER BY order_line_id
   Description: Individual line items for each order
   Key Fields: order_line_id, order_id, product_id, quantity, unit_price_eur

5. INVENTORY (23,409 records)
   Primary Key: inventory_id
   Engine: MergeTree() ORDER BY inventory_id
   Description: Real-time inventory tracking across all store locations
   Key Fields: inventory_id, product_id, store_id, quantity_available, last_updated

6. STORES (35 records)
   Primary Key: store_id
   Engine: MergeTree() ORDER BY store_id
   Description: Store location master data with operational information
   Key Fields: store_id, store_name, country, city, store_type, opening_date

7. CAMPAIGNS (4 records)
   Primary Key: campaign_id
   Engine: MergeTree() ORDER BY campaign_id
   Description: Marketing campaigns and promotional activities
   Key Fields: campaign_id, campaign_name, start_date, end_date, budget_eur

8. EUROPEAN_GEOGRAPHY (23 records)
   Primary Key: country_code
   Engine: MergeTree() ORDER BY country_code
   Description: European country and region reference data
   Key Fields: country_code, country_name, currency_code, vat_rate

9. FASHION_CALENDAR (25 records)
   Primary Key: calendar_id
   Engine: MergeTree() ORDER BY calendar_id
   Description: Fashion industry calendar events and seasonal markers
   Key Fields: calendar_id, event_name, event_date, season, importance

10. CUSTOMER_360_VIEW (Materialized View)
    Description: Real-time customer analytics view combining orders, preferences
    Source: Joins customers, orders, order_lines tables
    Refresh: Real-time via ClickHouse materialized view technology

================================================================================
                              DATA RELATIONSHIPS
================================================================================

FOREIGN KEY RELATIONSHIPS:
- orders.customer_id → customers.customer_id
- order_lines.order_id → orders.order_id  
- order_lines.product_id → products.product_id
- inventory.product_id → products.product_id
- inventory.store_id → stores.store_id
- customers.country → european_geography.country_code

CROSS-DATABASE INTEGRATIONS:
- orders integrate with eurostyle_finance.gl_journal_headers (perfect revenue match)
- customers integrate with eurostyle_hr for employee customer relationships
- inventory integrates with eurostyle_pos for real-time stock updates
